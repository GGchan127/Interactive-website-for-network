BGP — THE INTERNET PROTOCOL

BGP is the Exterior Gateway Protocol (EGP) that holds the Internet together. It routes between Autonomous Systems (AS) — collections of IP prefixes under a single routing policy.

ANIMATED — eBGP peering session between two Autonomous Systems
eBGP vs iBGP
eBGP: Between different ASes. Default TTL=1 (directly connected). Used between ISPs and enterprises.

iBGP: Within the same AS. Requires full-mesh or Route Reflector. Uses LOCAL-PREF for path selection.
PATH ATTRIBUTES (selection order)
1. Weight (Cisco-specific, higher=best)
2. LOCAL-PREF (higher=best, iBGP)
3. AS-PATH (shorter=best)
4. ORIGIN (IGP > EGP > ?)
5. MED (lower=best, hint to neighbour)
BASIC BGP CONFIG (Cisco IOS)
router bgp 100
 neighbor 203.0.113.2 remote-as 200
 network 10.0.0.0 mask 255.255.255.0
 no auto-summary

show bgp summary        ! Check neighbour states (Established?)
show ip bgp             ! Full BGP table
show ip bgp neighbors   ! Detailed session info
BGP uses TCP port 179 — reliable transport means BGP doesn't need its own reliability mechanism.
AS numbers: Public ASNs 1–64511 (IANA assigned) · Private ASNs 64512–65535
