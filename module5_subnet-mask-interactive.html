WHAT IS A SUBNET MASK?

An IP address is always split into two parts: the Network ID (which network — like a village name) and the Host ID (which device — like a house number). The Subnet mask tells you exactly where the split is.

The Subnet mask is a 32-bit value (written in dotted-decimal like an IP address). Where the mask has a 1 bit → Network portion. Where the mask has a 0 bit → Host portion.

The 1-bits always run consecutively from the left — you can never have a 0 between two 1s.

KEY: Subnet mask is AND-ed with the IP address to compute the Network ID instantly.

Example: 255.255.255.0 = 11111111.11111111.11111111.00000000 → Octets 1–3 = Network, Octet 4 = Host

THE AND OPERATION

To find the Network ID, AND every bit of the IP with the corresponding bit of the Subnet mask. Rule: 1 AND 1 = 1; anything with a 0 gives 0.

AND Truth Table:   0 AND 0 = 0  |  0 AND 1 = 0  |  1 AND 0 = 0  |  1 AND 1 = 1

SUBNET MASK BIT PATTERN

255 in any octet means "all 1s" = Network. 0 means "all 0s" = Host. The 1s always fill from the left.

DEFAULT SUBNET MASKS BY CLASS

Each IP class has a default mask matching its built-in N/H structure. These apply when no custom subnetting is used.

ClassFormatDefault Subnet MaskPrefixNet BitsHost BitsMax Hosts
AN.H.H.H255.0.0.0/882416,777,214
BN.N.H.H255.255.0.0/16161665,534
CN.N.N.H255.255.255.0/24248254
Max Hosts = 2h − 2   (h = host bits)
Subtract 2 because: 1 address reserved as Network Address (all host bits = 0) + 1 reserved as Broadcast Address (all host bits = 1). Neither can be assigned to a device.

SUBNETTING

Subnetting means "borrowing" bits from the Host portion to create smaller sub-networks — instead of one large network, you divide it into many smaller ones.

WHY SUBNET?

A Class C network has 254 hosts — all devices share the same broadcast domain, increasing traffic and reducing security.

Subnetting gives each department or floor its own isolated network.

HOW IT WORKS

The subnet mask gains more 1s (longer prefix) as bits are borrowed — e.g. Class A /8 → can borrow to /9, /10 ... /32

More borrowed bits → more subnets, but fewer hosts per subnet.

KEY FORMULAS

NUMBER OF SUBNETS
2n

n = borrowed bits. Each extra borrowed bit doubles the subnets.

HOSTS PER SUBNET
2h − 2

h = remaining host bits. Subtract 2 for Network and Broadcast addresses.

SUBNETS PER CLASS (Examples)
Class A: /9–/32

Class A: /9–/32  |  Class B: /17–/32  |  Class C: /25–/32

MASK FROM PREFIX
/n → count n ones

e.g. /26 = 11111111.11111111.11111111.11000000 = 255.255.255.192
