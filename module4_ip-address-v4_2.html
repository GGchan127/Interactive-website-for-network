WHAT IS AN IP ADDRESS?

An IP Address is a Logical Address — an address assigned to devices connected on a network. Because it is logical (not physical), an IP address can be changed or reconfigured. The result of setting an IP address is that devices can communicate with one another.

The easiest way to understand it is the village & house analogy used in the lesson:

An IP address is always made of two parts joined together — the Network ID (which network you belong to) and the Host ID (which specific device you are).

Real example: In a Wireshark capture of a visit to www.cisco.com, you can see the packet's Source IP (192.168.1.34) and Destination IP (23.199.41.25) — these addresses are how the data finds its way there and back.

IP ADDRESS FORMAT: IPv4

IPv4 has a total of 32 bits, divided into 4 groups (octets) of 8 bits each, separated by dots (.). Each octet's binary value is converted into a decimal number from 0–255.

Tip: The 8 bit positions have values 128, 64, 32, 16, 8, 4, 2, 1. Add up the positions where there is a "1". Example: 11000000 = 128 + 64 = 192. You can verify this with the Windows Programmer Calculator (BIN ↔ DEC).

IPv4 ADDRESS CLASSES

We can divide IP Addresses into classes — Class A, B, C, D, E — based on the value of the first octet. Class D is for multicast; Class E is reserved. The blue (N) = Network portion, gold (H) = Host portion.

Fig 3.1 — IPv4 address classes: first-octet ranges, Network/Host split, Public vs Private, and loopback.

A

Class A

First octet: 1 – 126  ·  pattern 0xxxxxxx

For very large networks. 1 network octet + 3 host octets.

NHHH
Private: 10.0.0.0 – 10.255.255.255Mask /8
B

Class B

First octet: 128 – 191  ·  pattern 10xxxxxx

For medium/large networks. 2 network octets + 2 host octets.

NNHH
Private: 172.16.0.0 – 172.31.255.255Mask /16
C

Class C

First octet: 192 – 223  ·  pattern 110xxxxx

For small networks (homes, small offices). 3 network octets + 1 host octet.

NNNH
Private: 192.168.0.0 – 192.168.255.255Mask /24
D

Class D — Multicast

224.0.0.0 – 239.255.255.255

Reserved for multicast — sending to a group of devices at once.

E

Class E — Reserved

240.0.0.0 – 255.255.255.254

Reserved for experimental / research use.

Loopback: 127.0.0.1 – 127.255.255.255 is reserved for loopback (your own device, "localhost") — that is why Class A stops at 126 and Class B starts at 128.
Public IP

A globally unique address routable on the Internet, assigned by your ISP. Example: a web server like www.cisco.com that the whole world can reach.

Private IP

Used inside local networks — not routable on the public Internet:

10.0.0.0 – 10.255.255.255 (Class A) 172.16.0.0 – 172.31.255.255 (Class B) 192.168.0.0 – 192.168.255.255 (Class C)

FINDING NETWORK ID & HOST ID

Once you know the class, you know where to split the address. Class A splits after 1 octet, Class B after 2, Class C after 3. Try the last two yourself, then reveal the answer.

10.16.127.10410.16.127.104A

195.31.72.123195.31.72.123C
172.29.16.100 172.29. 16.100 B
203.192.33.145 203.192.33. 145 C
How to decide the class: look at the first octet. 172 → 128-191 → Class B (split after 2 octets). 203 → 192-223 → Class C (split after 3 octets).

VIEW YOUR IP WITH ipconfig

IP addressing works at Layer 3 of the network model. On Windows, you can see your own device's IP settings using the Command Prompt command ipconfig /all.

Administrator: Command Prompt
C:\Users\user> ipconfig /all
 
Wireless LAN adapter Wi-Fi:
   Description . . . . . . : Intel(R) Wireless-N 7260
   Physical Address . . . : A0-A8-CD-17-36-F8
   DHCP Enabled . . . . . : Yes
   IPv4 Address . . . . . : 192.168.1.53 (Preferred)
   Subnet Mask . . . . . : 255.255.255.0
   Default Gateway . . . : 192.168.1.1
   DHCP Server . . . . . : 192.168.1.1
   DNS Servers . . . . . : 192.168.1.1
What to look for: IPv4 Address (your device's IP), Subnet Mask (defines the network/host split), Default Gateway (the router), and DNS Servers. Here 192.168.1.53 is a Class C private IP.
