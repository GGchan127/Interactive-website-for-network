// Link-State Routing
What is OSPF?

OSPF (Open Shortest Path First) is an open-standard Link-State Routing Protocol defined by RFC 2328, used as an Interior Gateway Protocol (IGP) within a single Autonomous System. Unlike distance-vector protocols, each router builds a complete topology map (LSDB) and independently runs Dijkstra's algorithm to compute the shortest path.

⚡ Fast Convergence
When a link fails, routers recalculate and install new paths within seconds — far faster than distance-vector protocols.
🗺️ Complete Topology Map
Every router holds the full network map (LSDB), not just next-hop distance and direction like RIP does.
∞ No Hop Count Limit
RIP caps at 15 hops. OSPF has no such constraint, making it suitable for large enterprise and ISP networks.
⚖️ ECMP Load Balancing
Traffic distributed across multiple equal-cost paths simultaneously using Equal-Cost Multi-Path routing.
🔐 Authentication
Supports MD5 and SHA authentication to prevent rogue routers from joining the OSPF domain.
📊 VLSM / CIDR Support
Supports variable-length subnet masks and route summarization to keep routing tables lean.
OSPF Cost Metric OSPF uses Cost as its metric: Cost = 100 Mbps ÷ Interface Bandwidth. For links faster than 100 Mbps (GigabitEthernet, 10G), adjust using auto-cost reference-bandwidth — otherwise all fast interfaces will have cost = 1 and OSPF cannot differentiate them.

📡 DR / BDR Election
Used on Multi-Access networks (Ethernet) to reduce flooding overhead

DR = Designated Router (highest priority wins)
BDR = Backup DR (second-highest)

Priority = 0 → excluded from election entirely
Equal priority → highest Router-ID wins

Point-to-point links do not use DR/BDR

// Areas & Hierarchy
OSPF Structure

OSPF uses a 2-Level Hierarchy — dividing the network into smaller Areas to reduce LSDB size and limit LSA flooding. Every Area must connect to Area 0 (the Backbone).

OSPF Multi-Area Topology — Area 0 (Backbone), Area 1 (Stub), Area 2 (NSSA)

// Link-State Advertisements
LSA Types

An LSA (Link-State Advertisement) is the routing information a router advertises to describe its local topology. Different LSA types serve different scopes and purposes within OSPF.

LSA TypeNameGenerated ByScopeDescription
Type 1Router LSAEvery routerIntra-AreaDescribes a router's links and their costs
Type 2Network LSADR on multi-accessIntra-AreaLists all routers on a broadcast segment
Type 3Summary LSAABRInter-AreaSummarizes prefixes from one Area into another
Type 4ASBR Summary LSAABRInter-AreaTells other areas where the ASBR is located
Type 5External LSAASBRAll Areas (not Stub)External routes injected via redistribution
Type 7NSSA External LSAASBR in NSSANSSA onlyConverted to Type 5 when leaving NSSA
E1 vs E2 External Routes E1 accumulates cost from the source (ASBR) all the way to the destination. E2 uses only the fixed metric set at the ASBR (default). Use E1 when multiple ASBRs exist — it accounts for internal path cost and selects the nearest ASBR automatically.

// Cisco IOS
OSPF Commands on Cisco IOS

Basic Configuration

ospf-basic-config.ios
! Enable OSPF process (process-id is local only — does not need to match) Router(config)# router ospf 1 ! Set Router-ID manually (always recommended — use a stable Loopback IP) Router(config-router)# router-id 1.1.1.1 ! Advertise networks (wildcard mask = inverse of subnet mask) Router(config-router)# network 10.0.12.0 0.0.0.3 area 0 Router(config-router)# network 1.1.1.1 0.0.0.0 area 0 ! Modern method (IOS 12.4+): enable OSPF directly on the interface Router(config-if)# ip ospf 1 area 0 ! Suppress Hello on interfaces with no OSPF neighbors (best practice) Router(config-router)# passive-interface default Router(config-router)# no passive-interface GigabitEthernet0/0

Interface Tuning

ospf-interface-tuning.ios
! Override the auto-calculated cost on an interface Router(config-if)# ip ospf cost 100 ! Adjust reference bandwidth to cover the fastest links (e.g. 10G) Router(config-router)# auto-cost reference-bandwidth 10000 ! Set DR/BDR priority (0 = never become DR or BDR) Router(config-if)# ip ospf priority 200 ! Tune Hello / Dead intervals (must match on both ends) Router(config-if)# ip ospf hello-interval 5 Router(config-if)# ip ospf dead-interval 20 ! Fix EXSTART issue caused by MTU mismatch Router(config-if)# ip ospf mtu-ignore ! Skip DR/BDR election on Ethernet P2P links Router(config-if)# ip ospf network point-to-point

Authentication (MD5)

ospf-authentication.ios
! Method 1: per-interface Router(config-if)# ip ospf authentication message-digest Router(config-if)# ip ospf message-digest-key 1 md5 MySecretKey123 ! Method 2: area-level (applies to all interfaces in the area) Router(config-router)# area 0 authentication message-digest Router(config-if)# ip ospf message-digest-key 1 md5 MySecretKey123

Verification Commands

ospf-verify.ios

Lab 2 — Multi-Area + Summarization + Redistribution

Advanced topology: Area 1 (Stub) → ABR → Area 0 → ASBR → BGP/Internet. Demonstrates route summarization at the ABR and BGP redistribution at the ASBR.

ABR Config — Route Summarization

R2 — ABR (Area 0 + Area 1 Stub)
R2(config)# router ospf 1 R2(config-router)# router-id 2.2.2.2 ! Interface facing Area 1 R2(config-router)# network 10.0.12.0 0.0.0.3 area 1 ! Interface facing Area 0 (Backbone) R2(config-router)# network 10.0.0.0 0.0.0.3 area 0 ! Summarize Area 1 routes (172.16.1-4.0/24 → 172.16.0.0/22) R2(config-router)# area 1 range 172.16.0.0 255.255.252.0 ! Make Area 1 Totally Stubby — minimizes LSA flooding R2(config-router)# area 1 stub no-summary ! All routers inside Area 1 must also configure: R4(config-router)# area 1 stub

ASBR Config — Redistribute BGP into OSPF

R3 — ASBR (BGP + OSPF)
! BGP toward ISP R3(config)# router bgp 65000 R3(config-router)# neighbor 203.0.113.1 remote-as 65001 ! OSPF on the ASBR R3(config)# router ospf 1 R3(config-router)# router-id 3.3.3.3 R3(config-router)# network 10.0.0.4 0.0.0.3 area 0 ! Redistribute BGP into OSPF as External Type 2 R3(config-router)# redistribute bgp 65000 subnets metric 100 metric-type 2 ! Inject default route into OSPF (advertise toward ISP) R3(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1 R3(config-router)# default-information originate always

// Troubleshooting
Troubleshooting & Best Practices

Common Issues

SymptomLikely CauseResolution
Neighbor stuck at INITHello/Dead interval mismatch or Area ID mismatchRun show ip ospf interface on both sides and compare
Stuck at EXSTART / EXCHANGEMTU mismatch between interfacesApply ip ospf mtu-ignore or align MTU
Neighbor up but route missingpassive-interface enabled or wrong network statementCheck show ip ospf database to confirm LSA exists
Route shows "O" but unreachableACL blocking traffic or lower-AD route overriding OSPFUse traceroute and check show ip route at each hop
Neighbor flapping repeatedlyDead interval too short or router CPU overloadedIncrease Dead interval to 40s+ or deploy BFD

Best Practices

Always set router-id manually. Use a stable Loopback IP. Never let OSPF pick one automatically — it will change if a higher-IP interface comes up.
Use passive-interface default. Start with passive-interface default, then selectively enable only on OSPF neighbor links with no passive-interface.
Tune reference-bandwidth. Set to match the fastest link in your network (e.g. auto-cost reference-bandwidth 10000 for 10G) so cost comparisons are meaningful.
Use area range on ABRs. Always summarize at ABR boundaries — fewer LSAs = smaller LSDBs, faster SPF, smaller routing tables across the domain.
Enable MD5 authentication on every production OSPF interface to prevent unauthorized routers from injecting false routing information.
Use P2P network type on WAN. ip ospf network point-to-point eliminates DR/BDR election on serial/Ethernet P2P links and speeds up adjacency.
⚠️ Warning Never run debug ip ospf on a production router with many neighbors. It can spike CPU usage enough to cause OSPF flaps across the entire network. Always run undebug all immediately after any debug session.
