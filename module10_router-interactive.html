A Router connects separate networks — it receives packets, consults its Routing Table to find the best path, and forwards the packet toward its destination, one hop at a time.

01
// What is a Router

WHAT IS A ROUTER?

A Switch forwards frames within a single network (LAN) using MAC addresses. A Router forwards packets between different networks using IP addresses. Where a Switch is Layer 2, a Router operates at Layer 3 — the Network Layer.

THE SWITCH (L2)
Works within one subnet. Forwards using MAC addresses. Broadcasts stay inside the LAN. Cannot connect different networks.
THE ROUTER (L3)
Connects multiple networks. Forwards using IP addresses. Stops broadcasts — each network segment is a separate broadcast domain.
THE RESULT
PC on 192.168.1.x can communicate with PC on 10.0.0.x only through a Router. The Router is the gateway between subnets.
💡 REAL-WORLD ANALOGY
Switch = Post Office within a city
Delivers letters within the same city using street addresses (MAC). Cannot send outside.
Router = International Courier Hub
Receives packages and forwards them to the correct country/city (network). Each hop brings the packet closer to its final destination.

02
// Forwarding Decision Process

HOW A ROUTER FORWARDS

When a packet arrives at a Router interface, the Router performs a 3-step decision: strip the Layer 2 frame, read the Layer 3 IP destination, look up the Routing Table, then re-encapsulate and forward.

① RECEIVE
Router receives the Ethernet frame on an interface. It strips the L2 header — the MAC addresses are only valid for one hop.
② LOOKUP
Router reads the IP destination from the packet header. It searches the Routing Table for the best matching route (longest prefix match).
③ FORWARD
Router re-encapsulates the packet in a new L2 frame (new src/dst MACs) and sends it out the correct interface. TTL is decremented by 1.

LONGEST PREFIX MATCH
  • Router picks the most specific route that matches
  • /24 beats /16 which beats /0 (default route)
  • Default route (0.0.0.0/0) = last resort — send anywhere unknown
  • No match and no default = packet dropped (ICMP unreachable)
ROUTE SOURCES
  • C — Connected: Router's own interfaces
  • S — Static: Manually configured by admin
  • O — OSPF: Learned via routing protocol
  • R — RIP: Learned via RIP protocol

// Routing Table Structure

ROUTING TABLE

Every router maintains a Routing Table — a database of known networks and how to reach them. Each row is a route, and together they describe the router's complete map of the network.

Router# show ip route
Codes: C - connected, S - static, R - RIP, O - OSPF, * - candidate default
 
C   192.168.1.0/24  is directly connected, GigabitEthernet0/0
C   10.0.0.0/24    is directly connected, GigabitEthernet0/1
S*  0.0.0.0/0      [1/0] via 203.0.113.1, GigabitEthernet0/2
                      [AD/Metric]
 
Router#

WHAT EACH COLUMN MEANS

  • Network / Mask: Destination network in CIDR notation
  • Next Hop: IP of next router (or "directly connected")
  • Interface: Which port to send the packet out of
  • AD/Metric: Administrative Distance / cost — lower = preferred
  • Source Code: C = connected, S = static, O = OSPF, R = RIP

ADMINISTRATIVE DISTANCE

  • 0 — Directly connected interface (most trusted)
  • 1 — Static route
  • 90 — EIGRP (Cisco proprietary)
  • 110 — OSPF (open standard)
  • 120 — RIP (older protocol)
  • 255 — Unknown / untrusted (never used)
Key rule:   If two routes exist to the same destination, the router uses the one with the lowest Administrative Distance. If AD is equal, the lowest Metric (cost) wins.

// Layer 2 vs Layer 3 at Each Hop

L2 CHANGES, L3 STAYS

This is the most important concept in routing: the IP addresses (L3) never change as a packet crosses networks. But the MAC addresses (L2) change at every hop — because MAC addresses are only meaningful on one physical segment.

HOP 1 — PC-A TO ROUTER
  • Src IP: 192.168.1.10 (unchanged)
  • Dst IP: 10.0.0.10 (unchanged)
  • Dst MAC: Router's Gi0/0 MAC
  • Src MAC: PC-A's MAC
  • TTL: 64
HOP 2 — ROUTER TO PC-B
  • Src IP: 192.168.1.10 (unchanged)
  • Dst IP: 10.0.0.10 (unchanged)
  • Dst MAC: PC-B's MAC ← CHANGED
  • Src MAC: Router's Gi0/1 MAC ← CHANGED
  • TTL: 63 (decremented by 1)
TTL (Time To Live):   Every router decrements the TTL by 1. If TTL reaches 0, the router drops the packet and sends an ICMP "Time Exceeded" message back to the source. This prevents packets from looping forever. Default TTL is usually 64 (Linux) or 128 (Windows).

// Router Commands & Verification

ROUTER COMMANDS

Use these commands to verify routing, test connectivity, and trace the path a packet takes through the network.

Command Prompt / Cisco IOS — Routing Verification
C:\Users\student> ping 10.0.0.10
 
Pinging 10.0.0.10 with 32 bytes of data:
Reply from 10.0.0.10: bytes=32 time=2ms TTL=63
Reply from 10.0.0.10: bytes=32 time=1ms TTL=63
 
Note: TTL=63 because router decremented it by 1 (started at 64)
 
C:\Users\student> tracert 10.0.0.10
 
Tracing route to 10.0.0.10 over a maximum of 30 hops:
  1    1 ms  192.168.1.1  [Router Gi0/0 — Default Gateway]
  2    2 ms  10.0.0.10   [Destination PC-B]
 
Trace complete.

COMMAND REFERENCE

ping <IP>
Test if destination is reachable. TTL in reply shows hop count used.
tracert / traceroute <IP>
Shows each hop (router) the packet passes through on its path.
show ip route
Display full routing table on a Cisco router. Shows all routes and their sources.
ip route <net> <mask> <nexthop>
Add a static route. E.g. ip route 172.16.0.0 255.255.0.0 192.168.1.254
ipconfig /all (Windows)
Shows Default Gateway IP — the router PC-A sends all non-local traffic to.
show interfaces
Show interface status, IP, MAC, and error counters on each router port.
