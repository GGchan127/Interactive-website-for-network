// Overview

FROM ONE IP, FOUR ANSWERS

When you are handed a single IP address with a prefix — for example 10.10.10.0/30 — you are expected to derive everything else about that subnet. There are three ways to get there: by writing it out in binary, by using a fast decimal shortcut, or by feeding it into a subnet calculator program. This module walks through all three.

① Network ID

The address that names the subnet. All host bits set to 0. It can never be assigned to a device.

② Broadcast IP

The last address in the subnet — all host bits set to 1. Reserved for broadcasting to every host.

③ Host Range

Every address between Network ID and Broadcast — these are the only addresses you can assign.

④ Host Count

How many usable addresses exist: 2ⁿ − 2, where n is the number of host bits.

Three methods:  1) bit type Binary  |  2) Decimal shortcut  |  3) use a Program / Subnet calculator. The binary method shows you why the shortcut works.

BINARY BASICS

Every octet is 8 bits. Each bit position carries a fixed weight: 128 · 64 · 32 · 16 · 8 · 4 · 2 · 1. To convert a decimal octet to binary, subtract weights from left to right: if the weight fits, write 1 and subtract; if not, write 0. Add the weights under the 1s and you get the decimal back.

// Step by step

THE METHOD, BY HAND

No calculator? Here is the exact sequence the lessons follow to solve any IP / prefix by hand.

1

Write the mask from the prefix

Put the prefix-many 1s from the left, fill the rest with 0s, then group into octets. Example: /30 → 11111111.11111111.11111111.11111100 → 255.255.255.252.

2

AND the IP with the mask → Network ID

Bit by bit: 1 AND 1 = 1, anything with a 0 gives 0. All host bits collapse to 0. That result is your Network ID.

3

Set all host bits to 1 → Broadcast

Take the Network ID and flip every host bit to 1. That is the Broadcast address — the highest address in the subnet.

4

Range = Network+1 … Broadcast−1

The usable host range is every address strictly between the Network ID and the Broadcast. First host = Network ID + 1; Last host = Broadcast − 1.

5

Count hosts with 2ⁿ − 2

Count the host bits (n = 32 − prefix). Usable hosts = 2ⁿ − 2. We subtract 2 for the Network ID and the Broadcast address, which can't be assigned.

// Putting it together

IT ALL FITS A NETWORK

The four examples aren't random — they're the addresses on a real topology. The /30 links join routers point-to-point; the /24 blocks are the LANs where end devices live.

The two router-to-router links use /30 subnets (just 2 usable addresses each — perfect for a point-to-point link), while each site LAN is a /24 with room for 254 devices.
