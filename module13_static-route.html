// The concept

WHAT IS ROUTING?

Routing is the process of forwarding a packet from one network to another through a Layer 3 device (a Router or a Layer-3 switch). Think of it like the postal system — the lesson uses exactly this analogy:

📮
Layer 3 device = the postman

A Router or L3 Switch reads the destination and decides where to forward the packet next.

✉️
Routed Protocol = IP = the address on the envelope

IP is the protocol being carried; the destination IP tells the router where the packet must end up.

🧭
Routing Protocol = the person who teaches the route

RIP, OSPF, IS-IS, EIGRP, BGP — these tell the router how to reach networks it isn't directly connected to.

Key distinction: a routed protocol (IP) is the data being delivered; a routing protocol (OSPF, EIGRP…) is how routers learn the delivery paths. Don't confuse the two.

THE ROUTING TABLE

Every router keeps a routing table — the list of destination networks it knows and how to reach each one. Three columns matter: where the route came from, the destination network, and the outgoing interface used to get there.

Source (how learned)Destination NetworkInterface Outgoing
C Connected192.168.1.0/24Fa0/0
L Learned10.10.10.0/24S0/0
L Learned172.29.16.0/24S0/1

If a destination network is not in the table, the router drops the packet — it simply doesn't know where to send it. So the whole job of routing is making sure every reachable network has an entry.

// Where routes come from

THREE WAYS TO FILL THE TABLE

Routes enter the table from three sources. The first is automatic; the other two are the two methods the lesson highlights — Static and Dynamic.

① Connected Route

The router's own directly-connected network IDs. Added automatically the moment an interface has an IP and comes up — no configuration needed.

② Static Route

The Network Admin manually defines each path. The route is fixed until someone changes it. This is the focus of this module.

③ Dynamic Route

The router runs a routing protocol and learns paths automatically. Protocols split into two families:

IGP — Interior Gateway Protocols (inside one organization)

// Trade-offs

STATIC vs DYNAMIC

Both fill the routing table, but they suit different networks. The lesson lays out the trade-offs clearly:

STATIC ROUTE
  • Admin sets the path — fully under your control
  • Saves CPU & memory — no protocol running
  • More secure — no route advertisements to intercept
  • Best for small networks
  • Doesn't scale — every route entered by hand
  • No auto-recovery — a failed link needs manual fixing
DYNAMIC ROUTE
  • Router chooses the path via the routing protocol
  • Scales to large networks
  • Auto-adapts — reroutes around failures
  • Uses more CPU & memory
  • Path not fixed — depends on the protocol's choice
  • More complex to secure and troubleshoot
Rule of thumb: small/stable networks and stub links → static. Large/changing networks → dynamic. Many real networks use both.

BUILD A STATIC ROUTE

A static route tells the router: "to reach this destination network, send packets to this next-hop (or out this interface)." The IOS syntax is ip route [network] [mask] [next-hop | exit-interface]. Fill in the fields — or load a scenario from the course topology — and watch the command and resulting table build live.

SCENARIOS:
⚠ Enter a valid network, mask, and next-hop (each octet 0–255).
HQ1 — IOS CLI
show ip route (static)
In show ip route, static routes appear with the code S, connected with C. A default route uses ip route 0.0.0.0 0.0.0.0 [next-hop] — "send anything I don't recognize this way."

THE TOPOLOGY

The scenarios above come from this network — the same one used across the module. Three routers (HQ2 · HQ1 · Branch1) joined by /30 WAN links, each with a /24 LAN. HQ1 sits in the middle, so it needs static routes to reach both end LANs.

Why HQ1 needs static routes: HQ1 is directly connected only to the two /30 links. To forward traffic to 192.168.1.0/24 it must be told "next-hop 10.10.10.2 (Branch1)," and to reach 172.29.16.0/24, "next-hop 10.10.10.5 (HQ2)."

TRUNK — HOW IT CONNECTS

Trunking is a switching topic (covered in the VLAN module), but it's worth recapping here because routing and trunking meet at the router-on-a-stick: a single trunk link can carry several VLANs to a router that routes between them. Here's the short version.

🔌
ACCESS PORT

Carries one VLAN, connects to an end device. Frames leave untagged.

🛣️
TRUNK PORT

Carries many VLANs over one link (switch↔switch, or switch↔router). Frames are tagged with their VLAN ID.

🥢
ROUTER-ON-A-STICK

One trunk to a router with sub-interfaces lets it route between VLANs — where trunking meets routing.
