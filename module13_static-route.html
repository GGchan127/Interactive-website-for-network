HOW STATIC ROUTES WORK

A static route is manually configured on a router. The router uses its routing table to decide where to forward each packet. Static routes have Admin Distance = 1 — the most trusted non-connected route.

ANIMATED — Packet following static route R1 → R2 → R3 → Destination
CONFIGURATION
ip route 172.16.0.0 255.255.255.0 10.1.0.2
! Default route (gateway of last resort)
ip route 0.0.0.0 0.0.0.0 10.0.0.1
! Floating static (backup, AD=150)
ip route 10.0.0.0 255.0.0.0 10.1.0.2 150
VERIFICATION
show ip route
show ip route static
show ip route 172.16.0.0
ping 172.16.0.1 source g0/0
traceroute 172.16.0.1

Default route: ip route 0.0.0.0 0.0.0.0 <next-hop> — matches ALL destinations not found in the routing table. Every router pointing to the Internet uses this.
Floating static: higher AD than the primary protocol — only used when the primary route disappears.
