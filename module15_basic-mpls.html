HOW MPLS WORKS

Instead of looking up full IP destinations at every router hop, MPLS prepends a short 32-bit label to packets at the network edge. Core routers swap labels — much faster than IP lookups.

ANIMATED — Label Push (PE-1) → Swap (P-1, P-2) → Pop (PE-2)
LER — Label Edge Router
At the MPLS edge. Ingress LER classifies packets into FECs and pushes a label. Egress LER removes (pops) the label before delivering to the customer.
LSR — Label Switch Router
Core MPLS router. Receives labelled packet, looks up the label in the LFIB, swaps it for a new label, and forwards — no IP lookup needed. Very fast.
MPLS VPN (L3VPN)
Service providers use MPLS + VRFs to carry multiple customer VPN instances over shared infrastructure. Each customer's traffic is completely isolated.
MPLS Label (32-bit): 20-bit label value | 3-bit EXP/TC (QoS) | 1-bit S (bottom-of-stack) | 8-bit TTL
LDP (Label Distribution Protocol): routers exchange label-to-prefix bindings with neighbours
FEC (Forwarding Equivalence Class): group of packets forwarded the same way (same label)
