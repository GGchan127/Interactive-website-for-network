WHAT IS ICMP?

ICMP (Internet Control Message Protocol) works alongside TCP/IP at Layer 3. Unlike TCP or UDP which carry application data, ICMP is used exclusively for reporting errors and sending diagnostic information about the network.

ERROR REPORTING
Reports errors that occur during TCP/IP protocol operation — for example, when a router can't reach a destination or a packet exceeds the maximum hop count.
CONNECTION INFO
Communicates status information about the network connection between devices — used to check whether a path to a destination is available and working.
DIAGNOSTICS
Powers the two essential network troubleshooting commands: "ping" (test reachability) and "traceroute / tracert" (map the path to a destination).

PING

Ping sends an ICMP Echo Request (Type 8) to a destination and waits for an ICMP Echo Reply (Type 0). It measures round-trip time and packet loss — the essential first step in network troubleshooting.

📨 ICMP TYPE REQUEST (Type 8)

SRC: 192.168.1.102

DST: 172.29.16.10 (www.basic.com)

The source sends 4 packets, each containing 32 bytes of data by default.

📩 ICMP TYPE REPLY (Type 0)

SRC: 172.29.16.10

DST: 192.168.1.102

The destination replies to every request, confirming it is reachable and responsive.

TTL — TIME TO LIVE

Each router along the path decrements the TTL by 1. When TTL hits 0, the packet is discarded and an ICMP "Time Exceeded" message is sent back — this prevents infinite loops.

TTL COUNTDOWN — Starting at 128 (Windows default)
Common starting TTL values:   Windows = 128  |  Linux/Mac = 64  |  Cisco IOS = 255  |  You can estimate hops from destination TTL: e.g. received TTL=125 → ~3 hops from a Windows host

USEFUL PING FLAGS

ping 8.8.8.8 -l 40
Set payload size to 40 bytes
ping 8.8.8.8 -t
Continuous ping until Ctrl+C
ping 8.8.8.8 -n 10
Send 10 echo requests
ping www.google.com
Ping by hostname (tests DNS too)

TRACEROUTE / TRACERT

Tracert (Windows) / traceroute (Linux) tests end-to-end connectivity by revealing every Layer-3 device the packet passes through — it maps the full path and measures latency at each hop.

HOW IT WORKS

  • Sends ICMP packets with TTL = 1, then 2, 3 … up to 30
  • Each router decrements TTL; when it hits 0 it sends back an ICMP "Time Exceeded" message
  • Tracert collects those replies to build the hop-by-hop map
  • Final destination replies with ICMP "Echo Reply" — trace complete

READING THE OUTPUT

  • Hop # — sequential router number
  • 3× latency values — three probes sent per hop (in ms)
  • Hostname — reverse-DNS name of the router
  • IP address — router's interface IP
  • * * * — router didn't respond (firewall drops ICMP)

TRACEROUTE BY OPERATING SYSTEM

🪟 WINDOWS
Command:
tracert yahoo.com
  • Uses ICMP Echo Request with increasing TTL
  • Max 30 hops by default
  • 3 probes per hop
  • Flags: tracert -h 50 yahoo.com (set max hops)
🐧 LINUX
Command:
traceroute yahoo.com
  • Uses UDP by default (port 33434+)
  • Add -I to use ICMP instead
  • Max 30 hops by default
  • Flags: traceroute -m 50 yahoo.com (set max hops)
🍎 macOS
Command:
traceroute yahoo.com
  • Same syntax as Linux (traceroute)
  • Uses UDP by default
  • Add -I for ICMP mode
  • Flags: traceroute -m 50 yahoo.com (set max hops)
OS Command Protocol (default) Force ICMP
🪟 Windows tracert ICMP Echo Request — (always ICMP)
🐧 Linux traceroute UDP (port 33434+) traceroute -I
🍎 macOS traceroute UDP (port 33434+) traceroute -I
Key difference:   Windows tracert always uses ICMP, so it may be blocked by firewalls that filter ICMP.  Linux/macOS traceroute uses UDP by default, which can pass through more firewalls — use -I to switch to ICMP mode.

ICMP MESSAGE TYPES

ICMP defines many message types, each identified by a numeric Type field. The most important ones for network troubleshooting are:

TypeNameDirectionUsed by
0Echo ReplyDestination → SourcePing — response to Type 8
3Destination UnreachableRouter → SourceError: host/port/network not reachable
5RedirectRouter → HostRouter tells host of a better route
8Echo RequestSource → DestinationPing — initiates the test
11Time ExceededRouter → SourceTTL reached 0 — used by traceroute
12Parameter ProblemRouter → SourceBad IP header field
30TracerouteSource → RouterInformation request (informational)
Key pair for ping:   Source sends Type 8 (Echo Request) → Destination replies with Type 0 (Echo Reply)  |  Key for tracert:   TTL expired triggers Type 11 (Time Exceeded) from each intermediate router
