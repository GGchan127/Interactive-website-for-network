01
// Why VLANs exist

THE FLAT-NETWORK PROBLEM

In a poorly designed network, every device sits in one enormous broadcast domain. When any host sends a broadcast, every other device must receive and process it — across every floor, every switch. As the network grows, this becomes a real problem.

📡
Broadcast storms

One broadcast reaches the whole network. Traffic floods everywhere with no boundary, wasting bandwidth.

🧠
Wasted CPU & memory

Every device burns CPU and memory processing broadcasts that were never meant for it.

🔓
Weak security

With everyone in the same segment, any host can reach any other — there's no isolation between groups.

The fix: break the single large broadcast domain into several smaller ones. That is exactly what a VLAN does — logically, without buying more switches.
02
// The concept

WHAT IS A VLAN?

A VLAN (Virtual LAN) divides one Broadcast Domain into several smaller ones on the same physical switch. Each VLAN behaves like a separate LAN: a broadcast inside VLAN 10 stays inside VLAN 10. One VLAN maps to one IP subnet. By default, every port on a Cisco switch belongs to VLAN 1.

INTERACTIVE — One switch, before & after VLANs
🌐
Network ID = the group

Each VLAN is its own subnet, e.g. VLAN 1 → 192.168.1.0/24, VLAN 2 → 192.168.2.0/24.

🚪
Smaller broadcast domains

A broadcast in one VLAN never crosses into another — less noise, less wasted CPU.

🛡️
Isolation by design

VLANs can't talk to each other without a router or Layer-3 switch — built-in segmentation.

VLAN RANGE

VLAN IDs run from 0 to 4095, but not all are usable. Usable IDs are 1–1001 and (on some platforms) 1006–4094. You can create or delete VLANs in 2–1001 and 1006–4094. Type an ID below to see where it lands.

// How ports join

VLAN MEMBERSHIP MODES

How does a switch decide which VLAN a port belongs to? Two membership modes appear in the lesson:

STATIC VLAN

The VLAN is decided by the port configuration. You manually assign each port to a VLAN — the most common method.

VOICE VLAN

One port carries both Voice and Data traffic on different VLANs — saving switch ports and cabling when an IP phone and a PC share a connection.

Big picture: A port operates in one of two modes — Access (carries a single VLAN, for end devices) or Trunk (carries many VLANs, between switches). Static & Voice are how an access port gets its VLAN.

// Carrying many VLANs

THE TRUNK PORT

VLANs usually span more than one switch. A Trunk Port carries traffic for more than one VLAN over a single link — it is the uplink between two switches, or to a router or server. Without a trunk you would need one cable per VLAN between the switches.

INTERACTIVE — Access link vs Trunk link
🔌
ACCESS PORT

Belongs to one VLAN. Connects to a single end device. Frames leave untagged.

🛣️
TRUNK PORT

Carries many VLANs. Connects switch↔switch (or to a router/server). Frames are tagged so the other side knows which VLAN each frame belongs to.

// Tagging frames

TRUNK ENCAPSULATION

When a frame crosses a trunk, the switch tags it with its VLAN ID so the receiving switch can sort it back into the right VLAN. There are two methods:

1 · ISL (Cisco)

Inter-Switch Link — a Cisco-proprietary method. It wraps the whole Ethernet frame: adds a 26-byte ISL header + a 4-byte CRC = 30 bytes of overhead.

ISL HEADER
26 bytes
ETHERNET FRAME
up to 1518 B
ISL CRC
4 bytes

Header fields: DA · TYPE · USER · SA · LENGTH · SNAP · HSA · VLAN ID (15 bits) · BPDU · INDX · RES

configure ISL trunk
Switch(config)# interface Fa 0/1
Switch(config-if)# switchport trunk encapsulation isl
Switch(config-if)# switchport mode trunk
2 · IEEE 802.1Q

The open standard (use this). Instead of wrapping, it inserts a 4-byte tag into the existing frame (between Source Address and Type/Len). The tag holds a 12-bit VLAN ID. Supports a native VLAN (untagged).

DEST ADDR
SRC ADDR
802.1Q TAG
4 bytes
TYPE/LEN
DATA
FCS

Tag = TPID (2 B) + TCI (2 B): User Priority (3 bits) · CFI (1 bit) · VLAN ID (12 bits)

configure 802.1Q trunk
Switch(config)# interface Fa 0/1
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# switchport mode trunk
Why 802.1Q wins: it's vendor-neutral, adds only 4 bytes (vs ISL's 30), and is the default on virtually all modern switches. ISL is legacy Cisco-only.
